# Heavy Validation Battery

> Document status: HISTORICAL EVIDENCE.
> Current authority: `scripts/factoryctl.py` and `docs/validation/canonical-real-infra-audit.md`.
> Runtime boundary: This is not the current factory rule; it records the historical heavy-validation battery shape.

One pilot is not enough to validate the factory. The battery must stress
different product shapes, risk levels, agent modes and review paths.

## Scenario Matrix

| Scenario | Goal | Expected Stress |
|---|---|---|
| Product Face SaaS | visible product with frontend/mobile states | Product Face proof, QA, independent review |
| Agentic Browser Workflow | autonomous agents using browser/tools/memory | agentic AI security, memory steward, handoff |
| Solana/Quasar Work Package | onchain architecture and Auditor route | Auditor, Codex Security, remote proof, human gate |
| Cloud Release R4 | production-like infra/deploy card | R4 human gate, rollback, cloud/IaC, monitoring |
| Public Repo Release | open-source docs/code publish | public safety, supply chain, AutoReview |
| Private Real Product Paper | real messy product paper | full source resolution, SOT, architecture, specialist review |

## Required Evidence Per Scenario

- source ledger or scenario paper;
- factory card;
- gate report;
- required worker packets;
- at least one critical review;
- Receipt Five or explicit not-done boundary;
- lesson learned;
- proposed factory change or rejected change.

## Pass Criteria

A scenario passes only when:

- required workers are triggered correctly;
- missing evidence blocks the right transition;
- public artifacts remain public-safe;
- high-risk surfaces trigger security specialists;
- Product Face appears for visible product work;
- onchain work triggers Auditor;
- R3/R4 work triggers human gate;
- the result records what remains unproven.

## Why This Is Better

The factory must survive ambiguity, different domains and real constraints.
Single-pilot validation proves the path exists; multi-context validation proves
the path has shape, pressure tolerance and useful failure modes.
